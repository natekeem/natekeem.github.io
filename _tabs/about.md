# About

## 이 블로그는 무엇인가

`태희의 커밋로그`는 Project AI Autoblog를 만들며 남기는 개발 기록입니다.

처음에는 AI로 수익형 블로그를 자동화하는 시스템을 바로 만들고 싶었습니다. 하지만 실제로 시작해 보니, 자동화보다 먼저 필요한 것은 글의 품질 기준, 수동 검수 흐름, 그리고 반복 가능한 작업 방식이었습니다.

그래서 이 블로그에는 완성된 결과보다 만들어 가는 과정을 기록합니다.

## 지금 만들고 있는 것

현재는 두 가지를 먼저 진행하고 있습니다.

1. 한국어 Dev Blog
   - 프로젝트 결정, 막힘, 도구 선택, 세팅 과정을 기록합니다.

2. 첫 번째 수익형 블로그 실험
   - 영어권 40대 이상 독자를 대상으로 한 The Smart Suburbanite를 준비합니다.
   - 자동 발행이 아니라 수동 검수와 품질 기준부터 실험합니다.

## AI를 어떻게 사용하는가

이 블로그의 글은 ChatGPT, Codex, Roo Code, Claude Code 같은 AI 도구의 도움을 받을 수 있습니다.

다만 원칙은 명확합니다.

```text
AI는 초안을 돕는다.
사람은 방향을 정하고 최종 검토한다.
자동화는 검증된 반복 작업에만 적용한다.
```

## 현재 원칙

- 수동 먼저, 자동화는 나중에.
- 품질이 수량보다 우선.
- 신뢰와 독창성이 트래픽보다 우선.
- AI 생성 콘텐츠는 사람 검수 없이 발행하지 않는다.
- 수익 과장, 가짜 전문성, 검증 없는 주장은 하지 않는다.
